# 06. 运算符

Aperio 使用 C 风格的运算符语法，而不是传统汇编的助记符。`r1 = r2 + r3` 比 `add r1, r2, r3` 更符合现代开发者的肌肉记忆。

## 算术运算符

| 运算符 | 含义           | 等价的 x86 助记符 |
|--------|----------------|-------------------|
| `+`    | 加法           | `add`             |
| `-`    | 减法 / 取负    | `sub` / `neg`     |
| `*`    | 乘法           | `imul` / `mul`    |
| `/`    | 除法           | `idiv` / `div`    |
| `%`    | 取模           | `idiv` / `div`    |

```text
r1 = r2 + r3
r1 = r2 - r3
r1 = r2 * r3
r1 = r2 / r3
r1 = r2 % r3
r1 = -r2          // 取负（一元减号）
```

### 有符号与无符号

Std-Strict 的寄存器是无类型的，但乘法、除法、取模的**操作本身**有区分。为了消除歧义，编译器会根据操作数的来源类型推断：如果两个操作数都来自有符号的内存加载或有符号字面量，就使用有符号运算；反之使用无符号。

在绝大多数情况下你不需要关心这一点，但如果你想强制指定，可以使用类型化的运算符变体（实现待定）。

## 位运算符

| 运算符 | 含义       |
|--------|------------|
| `<<`   | 左移       |
| `>>`   | 右移       |
| `&`    | 按位与     |
| `\|`   | 按位或     |
| `^`    | 按位异或   |
| `~`    | 按位取反   |

```text
r1 = r2 << 3
r1 = r2 >> r3
r1 = r2 & 0xFF
r1 = r2 | 0x80
r1 = r2 ^ r3
r1 = ~r2
```

### 关于右移

右移的行为同样依赖符号性：

- 有符号右移：高位填符号位（算术右移，x86 `sar`）
- 无符号右移：高位填 0（逻辑右移，x86 `shr`）

判断规则与乘除法一致。

## 比较运算符

比较运算符**不能独立出现**，只能作为 `if` 语句的条件使用。Std-Strict 没有像 C 那样的"比较结果布尔值"概念。

| 运算符 | 含义     |
|--------|----------|
| `==`   | 等于     |
| `!=`   | 不等于   |
| `<`    | 小于     |
| `>`    | 大于     |
| `<=`   | 小于等于 |
| `>=`   | 大于等于 |

```text
if (r1 == r2) goto(@equal)
if (r1 > 0)   goto(@positive)
if (r1 != 0)  goto(@nonzero)
```

详见 [09. 控制流](./09_control_flow.md)。

## 取地址运算符

`&` 在**一元位置**时表示取地址（与 C 的语义一致），在**二元位置**时表示按位与。两种用法不会冲突，编译器通过上下文区分：

```text
r1 = &G_BUFFER     // 一元 &：取地址
r2 = r3 & 0xFF     // 二元 &：按位与
```

详见 [08. 取地址运算符](./08_address_of.md)。

## 运算符优先级

从高到低：

1. 一元运算符：`-`（取负）、`~`（按位取反）、`&`（取地址）
2. `*` `/` `%`
3. `+` `-`
4. `<<` `>>`
5. `&`（按位与）
6. `^`
7. `|`
8. 比较运算符：`==` `!=` `<` `>` `<=` `>=`

可以使用圆括号 `()` 强制指定求值顺序：

```text
r1 = (r2 + r3) * r4
r1 = r2 & (0xFF << 8)
```

## 不支持的运算符

Std-Strict **不提供**以下高级语言中常见的运算符：

- 逻辑运算符 `&&`、`||`、`!`（用显式的条件跳转替代）
- 复合赋值 `+=`、`-=`、`*=`（写 `r1 = r1 + r2` 即可，编译器会优化）
- 自增自减 `++`、`--`
- 三元运算符 `? :`
- 指针解引用 `*p`（用 `mem.i32[p]` 语法）
