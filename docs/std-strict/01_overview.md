# 01. 概述

Std-Strict 是 Aperio 的中间表示层（IR），介于高级的 Loose 模式和具体硬件的 Native-Strict 模式之间。它使用 `.ap` 作为文件扩展名。

## 定位

Std-Strict 的定位是一种**与架构无关的底层语言**。它保留了汇编的所有本质特征——显式的寄存器、显式的内存访问、显式的调用约定——但屏蔽了各个硬件平台的琐碎差异（物理寄存器数量、寻址模式、指令编码等）。

你可以把它类比为：

- LLVM IR 的三地址码形式
- WebAssembly 的局部变量（Locals）模型
- CUDA PTX 的虚拟寄存器模型

Std-Strict 最终会由编译器后端降级（lowering）到具体的目标架构——x86、ARM、RISC-V、Wasm 或 PTX。

## 设计原则

### 无限虚拟寄存器

Std-Strict 提供 `r0` 到 `r999` 共 1000 个虚拟寄存器。物理寄存器分配、溢出到栈（register spilling）等工作全部由编译器后端完成，你不需要关心目标架构到底有 16 个还是 32 个寄存器。

### 显式优于隐式

所有可能产生副作用的操作都必须显式写出来：

- 内存读写必须标明字节宽度和符号性（`mem.i32` / `mem.u8`）
- 函数必须声明它使用的参数寄存器、返回值寄存器和破坏的寄存器
- 控制流跳转使用显式的标签和 `goto`，没有隐藏的分支

### 无隐式栈帧

Std-Strict 不暴露 `sp` / `bp` 寄存器。栈内存通过内置的 `stack.alloc` 原语分配，函数返回时自动释放。这一方面避免了手写汇编时栈不平衡的问题，另一方面也让代码能无损地编译到 WebAssembly 这种没有硬件栈指针的目标上。

## 与其他层的关系

```
┌─────────────────────────────────────────────┐
│  Loose Mode (*.apo)                         │
│  - 结构化控制流 (if / while / for)           │
│  - 自动寄存器分配                             │
└──────────────────┬──────────────────────────┘
                   │  lowering
                   ▼
┌─────────────────────────────────────────────┐
│  Std-Strict (*.ap)    ← 本文档               │
│  - 虚拟寄存器 r0-r999                        │
│  - 显式 ABI 契约                             │
│  - 标签 + goto 控制流                         │
└──────────────────┬──────────────────────────┘
                   │  instruction selection
                   ▼
┌─────────────────────────────────────────────┐
│  Native-Strict (*.x86.ap / *.arm.ap / ...)  │
│  - 物理寄存器 r0-r15                         │
│  - 目标架构指令                               │
└─────────────────────────────────────────────┘
```

## 什么时候使用 Std-Strict

- 需要精确控制寄存器使用，但不想被具体架构绑死
- 编写需要跨 x86 / ARM / Wasm 的底层库
- 作为编译器的中间目标
- 学习或分析真实的汇编代码行为

如果你只需要快速写一段逻辑而不在意底层细节，使用 Loose 模式（`.apo`）会更合适。
