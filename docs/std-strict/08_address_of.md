# 08. 取地址运算符

`&` 在一元位置时用于获取一个符号的内存地址。它是 Aperio 里唯一用来把"名字"转成"地址"的桥梁。

## 语法

```
& <name>
```

其中 `<name>` 是一个已声明的符号，可以是：

- 全局常量（`val`）
- 可变全局变量（`var`）
- 函数名

## 取全局数据的地址

```text
val MSG: u8[] = "Hello"
var BUFFER: u8[1024]

pub fn demo() -> r0 uses (r1, r2) {
    r1 = &MSG          // r1 = MSG 的首字节地址
    r2 = &BUFFER       // r2 = BUFFER 的首字节地址
    r0 = 0
}
```

在 x86 后端上，这通常会被翻译成 `lea` 指令（或带符号重定位的 `mov`）。在 Wasm 后端上，会被翻译成对应数据段的线性内存偏移。

## 取函数的地址

可以取一个函数的地址，得到的是一个函数指针：

```text
pub fn add(r1, r2) -> r0 {
    r0 = r1 + r2
}

pub fn demo() -> r0 uses r9 {
    r9 = &add                        // r9 = add 函数的入口地址
    r9(r1 = 10, r2 = 20)             // 通过 r9 间接调用 add（r0 = 30）
}
```

详见 [10. 函数](./10_functions.md) 中关于间接调用的说明。

## 不能取的东西

以下情况**不能**使用 `&`：

- 寄存器：`&r1` 非法，寄存器本身不是内存对象
- 编译期常量：`&MAX_SIZE` 非法，`const` 只是编译期数值，在运行时不占内存
- 字面量：`&42` 非法
- 结构体字段：`&Point.x` 非法（字段偏移是编译期数值，见 [16. 结构体](./16_structs.md)）

如果你需要临时获取一段可寻址内存，应该使用 [`stack.alloc`](./17_stack.md)。

## 与按位与 `&` 的区分

二元位置的 `&` 是按位与运算符，语法上和取地址完全不冲突：

```text
r1 = &MSG              // 一元 & —— 取地址
r2 = r3 & 0xFF         // 二元 & —— 按位与
r4 = &MSG & 0xFFF      // 先取地址，再按位与（合法）
```

编译器通过 `&` 左右两边是否有操作数来判断。
