# 23. 宏系统

宏是**在编译期展开成代码**的元编程工具。Std-Strict 的宏系统故意保持简约——叫它 "Level A"：够用，不上瘾。

## 两种来源

| 来源       | 定义方                                    | 调用形式              | 用户可新增？ |
|------------|-------------------------------------------|-----------------------|--------------|
| 用户宏     | `.ap` 源码里的 `macro name!(...) { ... }` | `name!(...)`          | ✅           |
| 内置宏     | 编译器自带（声明上带 `#[macro]`）         | `name!(...)`          | ❌           |

**共同点**：

- 调用时都带 `!` 后缀，与函数调用明显区分
- 展开位置目前**只允许语句位置**（不在表达式右侧）
- 命名空间和 `import` 系统相同（`io::println!` 需要 `import "std/io" as io`）
- 参数是**位置参数**（不使用 `=` 命名形式）

## 用户宏语法

```
macro <name>!(<param1>: <frag1>, <param2>: <frag2>, ...) {
    <模板体>
}
```

- `<name>` 是宏名（小写蛇形）
- 每个参数带**片段类型**（fragment kind），编译器按片段类型解析
- 模板体里用 `$<param>` 引用参数，编译器把它们替换为展开时的实际内容

例：

```rust
// macros.ap
pub macro for_range!($i: reg<i64>, $start: expr, $end: expr, $body: block) {
    $i = $start
@__loop:
    if ($i >= $end) goto(@__end)
    $body
    $i = $i + 1
    goto(@__loop)
@__end:
}
```

使用：

```rust
import "./macros" as m

pub fn sum_0_to_100() -> (r0: i64) uses r1 {
    r0 = 0i64
    m::for_range!(r1, 0i64, 100i64, {
        r0 = r0 + r1
    })
}
```

### 片段类型

| 片段            | 匹配                                    |
|-----------------|-----------------------------------------|
| `expr`          | 任意表达式（`r1 + 2`、`&FOO`、`mem.i64[r3]` 等） |
| `reg`           | 任意寄存器名或其别名（`r1`、`f5`、`count`、`tmp_buffer`） |
| `reg<T>`        | 指定类型的寄存器或别名（`reg<i64>` 只接整数槽，`reg<f32>` 只接浮点槽） |
| `block`         | `{ ... }` 包围的语句块                 |
| `literal`       | 字面量（整数、浮点、字符串等，编译期可确定） |
| `ident`         | 标识符（函数名、符号名等）              |

不支持变长匹配（`$($x:expr),*`）和模式递归——Level A 不干这事。需要的话未来升级 Level B/C 再说。

### 卫生性（Hygiene）

**自动**：

- 宏内部出现的**新**标签（`@name`）按约定形如 `@__xxx`（双下划线前缀），编译器在每次展开时给它们打唯一后缀，避免和调用点的标签冲突
- 宏参数 `$x` 在模板体里按"已解析的 AST 节点"替换，不是纯文本替换——所以 `$x + 1` 遇到 `$x = a * b` 时不会产生优先级问题

**手动**：

- 宏外的寄存器是全局共享的。宏内写 `r3 = ...` 会写到调用方的 `r3`——想要"宏内部临时寄存器"目前得让调用方在 `uses` 里声明
- 宏展开后的寄存器写入同样要通过调用方函数的 `uses` 检查
- `reg` / `reg<T>` 片段匹配到的**别名**会被解析为底层槽再参与展开——宏作者看到的是"底层槽 + 当前类型"，可读性层在展开前就被剥离。如果调用点传入的是签名级或文件级别名，展开后的代码会自动用该别名书写，保持严格替换规则一致

### 模板体规则

- 模板体是**语句序列**——和函数体里合法的写法一致
- 参数 `$x` 可以出现在任何"片段类型规定的位置"
- 标签、控制流、内存访问、函数调用都可以用
- `$x` 不能出现在"宏定义不接受这种片段"的位置（编译器会按片段类型约束检查）

### 可见性

`pub macro name!` 跨模块可用（通过 `import`）。未加 `pub` 的宏仅在定义它的模块内可见。

## 内置宏

编译器内置宏用 `#[macro]` 属性声明（仅编译器自身能写）：

```rust
#[macro] pub macro println!(fmt: literal, ..args)
#[macro] pub macro assert!(cond: expr, fmt: literal = "", ..args)
#[macro] pub macro dbg!(expr: expr)
```

内置宏的特殊能力**高于**用户宏：

- 可以解析字符串字面量的**格式占位符**（如 `{d}` / `{s}`）
- 可以 lazy 求值（如 `assert!` 的条件在 release 下整体剥离）
- 可以捕获参数的**源码文本**（如 `dbg!` 打印 `"r1 + r2 = 15"`）
- 可以接收变长参数（`..args`）

清单：

| 模块         | 内置宏                                              |
|--------------|-----------------------------------------------------|
| `std/io`     | `println!` / `print!` / `eprintln!` / `eprint!` / `dbg!` |
| `std/debug`  | `assert!` / `panic!` / `static_assert!`             |

详见各自模块章节。

## 宏 vs `inline` 函数

| 需求                                  | 选哪个                     |
|---------------------------------------|----------------------------|
| 零调用开销但还要 ABI / 类型检查       | `inline fn`                |
| 参数要在编译期作为字面量字符串解析     | 内置宏（`println!`）        |
| 参数要 lazy 求值                       | 用户宏 / 内置宏             |
| 参数是一个"代码块"而非一个值           | 用户宏（`block` 片段）      |
| 想要编译期 `if` 决定展开哪段代码       | 结合 `#[cfg(...)]`（见 [37](./37_cfg.md)） |

一般原则：能写 `inline fn` 就写 `inline fn`——它更类型安全、IDE 友好。必须编译期做手脚才上宏。

## 示例：常用的几种用户宏

### 临时变量交换

```rust
pub macro swap!($a: reg, $b: reg, $tmp: reg) {
    $tmp = $a
    $a = $b
    $b = $tmp
}

// 调用方要在 uses 里把 tmp 加进去
fn demo(r1: i64, r2: i64) -> (r0: i64) uses r3 {
    m::swap!(r1, r2, r3)
    r0 = r1 - r2
}
```

### 条件赋值（配合 select）

```rust
pub macro clamp!($out: reg<i64>, $x: expr, $lo: expr, $hi: expr) {
    $out = core::select($x < $lo, $lo, $x)
    $out = core::select($out > $hi, $hi, $out)
}

// 调用方需要 import "std/core" as core
fn demo(r1: i64) -> (r0: i64) {
    m::clamp!(r0, r1, 0i64, 100i64)
}
```

注意宏展开后用到了 `core::select`——宏**不自带 import**，调用方仍需 `import "std/core" as core`。未来可能加 `macro` 内的 `use` 声明，目前需要调用方配合。

## 限制

- 宏**只能出现在语句位置**，不能当表达式用——想要"表达式宏"（如 `max!(a, b) + 1`）暂不支持
- 没有变长匹配 / 递归宏（Level A 的核心取舍）
- 没有 `proc_macro` / 编译期计算自由函数的能力
- 不能跨宏边界调试单步——展开点在源码里看得到，展开体在另一处
