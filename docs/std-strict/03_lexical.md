# 03. 词法

本章描述 Aperio 源码最基础的构成单元：注释、标识符和字面量。

## 注释

Aperio 支持两种注释风格，语义与 C/Rust 完全一致。

### 行注释

```text
// 这是一个行注释
r1 = r2 + r3  // 也可以写在行尾
```

### 块注释

```text
/*
  这是多行块注释。
  块注释可以嵌套：
  /* 这里是内层注释 */
*/
```

块注释支持嵌套，这和 C 不同，但和 Rust 一致。

## 标识符

标识符用于命名函数、全局变量、标签、结构体字段等。规则如下：

- 首字符：字母（`A-Z` / `a-z`）或下划线 `_`
- 后续字符：字母、数字或下划线
- 区分大小写
- 不能与保留字冲突

```text
// 合法
my_var
_internal
MAX_SIZE
Point2D
__module_private

// 非法
2fast         // 数字开头
hello-world   // 包含连字符
fn            // 保留字
```

标签名需要额外以 `@` 开头，详见 [09. 控制流](./09_control_flow.md)。

## 关键字

以下是 Aperio 的保留关键字，**不能**作为普通标识符（函数名、变量名、字段名等）使用。

### 声明与定义

| 关键字    | 用途                              |
|-----------|-----------------------------------|
| `fn`      | 声明函数                          |
| `struct`  | 声明结构体布局                    |
| `const`   | 声明编译期常量                    |
| `val`     | 声明只读数据段                    |
| `var`     | 声明可变数据段                    |
| `extern`  | 声明外部函数（FFI）               |
| `import`  | 引入其他模块                      |
| `as`      | `import ... as` 的别名语法        |

### 修饰符

| 关键字    | 用途                              |
|-----------|-----------------------------------|
| `pub`     | 模块外可见                        |
| `export`  | 符号在链接器层面可见              |
| `inline`  | 内联展开                          |

### 控制流

| 关键字    | 用途                              |
|-----------|-----------------------------------|
| `if`      | 条件判断（仅配合 `goto` 使用）    |
| `goto`    | 跳转                              |
| `return`  | 从函数返回                        |

### 函数签名子句

| 关键字    | 用途                                |
|-----------|-------------------------------------|
| `uses`    | 声明函数临时使用的寄存器（scratch） |

### 基础类型

`i8`, `i16`, `i32`, `i64`, `u8`, `u16`, `u32`, `u64`, `f32`, `f64`

### 内置命名空间

下面的名字作为**顶层前缀**被保留，你可以自己定义同名局部标识符，但不能定义与它们同名的顶层符号：

| 名字      | 用途                                       |
|-----------|--------------------------------------------|
| `mem`     | 内存访问：`mem.i32[...]` 等                |
| `stack`   | 栈管理：`stack.alloc(...)`                 |
| `syscall` | 系统调用                                   |
| `std`     | 内置标准库命名空间（`std::println` 等）    |

### 保留标识符

以下名字虽然不是传统意义上的"关键字"，但在语言中有特殊含义，不能被重新定义：

- **寄存器名**：`r0` 到 `r999`
- **栈参数占位**：`stack[N]` 形式中的 `stack`（与内置命名空间同名）
- **保留待用**：`match`, `loop`, `continue`, `break`, `let`, `use`, `trait`, `impl`, `mod`, `self`, `Self`, `where`, `unsafe`, `async`, `await`（预留给未来可能的语言扩展，目前不能使用）

## 整数字面量

支持四种进制前缀：

| 进制 | 前缀    | 示例            |
|------|---------|-----------------|
| 十进制 | （无） | `42`, `1000`    |
| 十六进制 | `0x`   | `0xFF`, `0xDEADBEEF` |
| 八进制 | `0o`   | `0o755`         |
| 二进制 | `0b`   | `0b1010_1100`   |

字面量中可以使用下划线 `_` 作为视觉分隔符，对值本身没有任何影响：

```text
r1 = 1_000_000
r2 = 0xFFFF_FFFF
r3 = 0b1111_0000_1010_0101
```

### 负数

负数写法就是前面加一个 `-`：

```text
r1 = -42
r2 = -0x10
```

## 浮点数字面量

浮点数遵循 IEEE 754，格式与 C 基本一致：

```text
r1 = 3.14
r2 = 1.0e-5
r3 = 6.022e23
r4 = .5        // 合法，等价于 0.5
r5 = 2.        // 合法，等价于 2.0
```

浮点字面量默认类型为 `f64`，在赋值给特定位宽的内存位置时会按需转换。

## 字符字面量

单个字符用单引号括起来，值为该字符的 ASCII 码（`u8` 类型）：

```text
r1 = 'A'       // 等价于 r1 = 65
r2 = '\n'      // 等价于 r2 = 10
r3 = '\0'      // 等价于 r3 = 0
```

支持的转义序列：

| 序列     | 含义         |
|----------|--------------|
| `\n`     | 换行 (0x0A)  |
| `\r`     | 回车 (0x0D)  |
| `\t`     | 制表符 (0x09)|
| `\0`     | 空字符 (0x00)|
| `\\`     | 反斜杠       |
| `\'`     | 单引号       |
| `\"`     | 双引号       |
| `\xHH`   | 十六进制字节 |

## 字符串字面量

字符串字面量只能出现在数据段定义中，不能直接赋值给寄存器。详见 [15. 字符串](./15_strings.md)。

```text
val GREETING: u8[] = "Hello"
val C_STR:    u8[] = c"Hello\0"
```
