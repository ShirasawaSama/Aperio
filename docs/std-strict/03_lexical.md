# 03. 词法

本章描述 Aperio 源码最基础的构成单元：注释、标识符、关键字和字面量的**词法**部分。字面量的**类型**规则另见 [06. 字面量](./06_literals.md)。

## 注释

Aperio 支持两种注释风格，语义与 C/Rust 完全一致。

### 行注释

```rust
// 这是一个行注释
r1 = r2 + r3  // 也可以写在行尾
```

### 块注释

```rust
/*
  这是多行块注释。
  块注释可以嵌套：
  /* 这里是内层注释 */
*/
```

块注释支持嵌套，这和 C 不同，但和 Rust 一致。

## 标识符

标识符用于命名函数、全局变量、标签、结构体字段等。规则：

- 首字符：字母（`A-Z` / `a-z`）或下划线 `_`
- 后续字符：字母、数字或下划线
- 区分大小写
- 不能与保留字冲突

```rust
// 合法
my_var
_internal
MAX_SIZE
Point2D
__module_private

// 非法
2fast         // 数字开头
hello-world   // 包含连字符
fn            // 保留字
```

标签名以 `@` 开头，详见 [11. 控制流](./11_control_flow.md)。

## 关键字

以下是 Aperio 的保留关键字，**不能**作为普通标识符使用。

### 声明与定义

| 关键字    | 用途                              |
|-----------|-----------------------------------|
| `fn`      | 声明函数                          |
| `const`   | 声明编译期常量 / 标记 `const fn`  |
| `val`     | 声明只读数据段                    |
| `var`     | 声明可变数据段                    |
| `struct`  | 声明结构体布局                    |
| `type`    | 声明类型别名（主要给函数指针用）  |
| `alias`   | 声明寄存器别名                    |
| `macro`   | 声明用户宏                        |
| `extern`  | 声明外部函数（FFI）               |
| `import`  | 引入其他模块                      |
| `as`      | `import ... as` / 类型转换        |

### 修饰符

| 关键字    | 用途                              |
|-----------|-----------------------------------|
| `pub`     | 模块外可见                        |
| `export`  | 符号在链接器层面可见              |

> 注：`inline`、`noreturn`、`cold`、`naked`、`section`、`align`、`hot`、`export_name`、`builtin`、`cfg`、`cfg_attr` 不是关键字，而是**属性**——写成 `#[inline]`、`#[cfg(...)]` 等。详见 [14. 函数属性](./14_function_attrs.md) 和 [37. 条件编译](./37_cfg.md)。

### 控制流

| 关键字    | 用途                              |
|-----------|-----------------------------------|
| `if`      | 条件判断（仅配合 `goto` 使用）    |
| `goto`    | 跳转                              |
| `return`  | 从函数返回                        |

### 函数签名子句

| 关键字    | 用途                                |
|-----------|-------------------------------------|
| `uses`    | 声明函数临时使用的寄存器（scratch） |

### 基础类型

`i8`, `i16`, `i32`, `i64`, `u8`, `u16`, `u32`, `u64`, `f32`, `f64`, `bool`, `void`

### 字面量值

| 关键字    | 值                               |
|-----------|----------------------------------|
| `true`    | 布尔真                           |
| `false`   | 布尔假                           |

### 保留标识符

以下名字不是传统意义上的关键字，但在语言中有特殊含义，不能被重新定义：

- **寄存器名**：`r0` 到 `r999`、`f0` 到 `f999`
- **栈参数占位**：签名里的 `stack[N]`
- **内置命名空间前缀**：`mem`（内存访问 `mem.<T>[...]`）
- **保留待用**：`match`, `loop`, `continue`, `break`, `let`, `use`, `trait`, `impl`, `mod`, `self`, `Self`, `where`, `unsafe`, `async`, `await`（预留给未来可能的语言扩展）

## 整数字面量

支持四种进制前缀：

| 进制     | 前缀 | 示例                   |
|----------|------|------------------------|
| 十进制   | （无）| `42`, `1000`           |
| 十六进制 | `0x` | `0xFF`, `0xDEADBEEF`   |
| 八进制   | `0o` | `0o755`                |
| 二进制   | `0b` | `0b1010_1100`          |

字面量中可以使用下划线 `_` 作为视觉分隔符，对值本身没有任何影响：

```rust
r1 = 1_000_000i64
r2 = 0xFFFF_FFFFu32
r3 = 0b1111_0000_1010_0101u16
```

### 类型后缀

整数字面量**通常需要**带后缀：`42i32`、`100u64`、`0xFFu8`。后缀决定了字面量的类型；省略时按上下文推导（见 [06. 字面量](./06_literals.md)）。

### 负数

负数写法就是前面加一个 `-`：

```rust
r1 = -42i32
r2 = -0x10i64
```

## 浮点字面量

浮点字面量必须包含小数点或指数部分（区别于整数字面量）：

```rust
f1 = 3.14f64
f2 = 1.0e-5f64
f3 = 6.022e23f64
f4 = 0.5f32
```

后缀 `f32` / `f64` 决定类型；无后缀时按上下文推导。

## 字符字面量

单个字符用单引号括起来，类型为 `u8`（ASCII 单字节）：

```rust
r1 = 'A'       // = 65u8
r2 = '\n'      // = 10u8
r3 = '\0'      // = 0u8
```

支持的转义序列：

| 序列     | 含义         |
|----------|--------------|
| `\n`     | 换行 (0x0A)  |
| `\r`     | 回车 (0x0D)  |
| `\t`     | 制表符 (0x09)|
| `\0`     | 空字符 (0x00)|
| `\\`     | 反斜杠       |
| `\'`     | 单引号       |
| `\"`     | 双引号       |
| `\xHH`   | 十六进制字节 |

## 字符串字面量

字符串字面量只能出现在数据段定义中（或宏的 `literal` 片段参数中），不能直接赋值给寄存器。详见 [19. 字符串](./19_strings.md)。

```rust
val GREETING: u8[] = "Hello"           // 不含结尾 \0
val C_STR:    u8[] = c"Hello"          // 末尾自动追加 \0
```

## 布尔字面量

```rust
r1 = true                              // bool
r2 = false                             // bool
```

`bool` 仅用于 `if` 条件、`std/core::select` 的条件参数和函数签名里。`bool` 和整数**不自动互转**，需要显式比较或转换。
