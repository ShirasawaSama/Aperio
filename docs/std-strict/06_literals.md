# 06. 字面量

字面量（literal）是源码里写死的常量值——整数、浮点、字符、字符串、布尔。本章讲清楚它们的**类型**是如何确定的，以及怎么用后缀显式指定。

## 整数字面量

### 书写形式

```rust
42              // 十进制
0xFF            // 十六进制（大小写均可：0xff / 0xFF）
0b1010_1100     // 二进制
0o755           // 八进制
1_000_000       // 下划线分隔符（仅视觉作用）
```

下划线可以出现在任意两个数字之间（不能在开头、结尾、紧挨基数前缀）。

### 类型后缀

```rust
42i8     42i16    42i32    42i64
42u8     42u16    42u32    42u64
0xFFu8
0b1010_1100u16
```

后缀直接贴在数字后面，没有空格。

### 类型推导

**无后缀**的整数字面量按以下优先级确定类型：

1. **签名 / 赋值目标驱动的显式上下文**

   ```rust
   mem.i32[r1] = 42             // 42 是 i32
   takes_i64(r1 = 42)           // 42 是 i64（假设签名参数是 i64）
   const MAX: u8 = 200          // 200 是 u8
   ```

2. **算术表达式里跟随另一个操作数**

   ```rust
   r1: u8
   r2 = r1 + 1                  // 1 被推导为 u8（跟随 r1）
   r3 = r1 - 2 + 3              // 2 和 3 都是 u8
   ```

   两端都是字面量时，默认取 `i64`：

   ```rust
   r2 = 1 + 2                   // 被推导为 i64
   ```

3. **没有任何线索**时默认 `i64`：

   ```rust
   r1 = 42                      // r1 的当前类型 = i64
   ```

### 溢出是编译错

字面量**在编译期**就会被按目标类型范围检查，超出即报错：

```rust
const X: u8 = 256              // 编译错：256 超过 u8 最大 255
const Y: i8 = 200              // 编译错：200 > 127
const Z: i8 = -129             // 编译错：-129 < -128
const W: u32 = 0xFFFF_FFFF_FF  // 编译错：位太多
```

## 浮点字面量

### 书写形式

```rust
3.14            // 有小数点
1.0e10          // 科学计数法
1.5e-3          // 带负指数
.5              // 开头省略 0
0.              // 末尾省略
```

### 类型后缀

```rust
3.14f32
3.14f64
1.0e10f64
```

### 类型推导

规则同整数字面量：上下文驱动 → 跟随另一个操作数 → 默认 `f64`。

```rust
f1: f32
f2 = f1 + 1.5                  // 1.5 被推导为 f32

f3 = 3.14                      // f3 的当前类型 = f64
```

浮点字面量**不能**自动变成整数，反之亦然——要转换必须走 `std/convert`。

## 字符字面量

```rust
'A'             // u8，值 = 65
'\n'            // u8，值 = 10
'\x7F'          // u8，值 = 127
'\''            // u8，单引号
'\\'            // u8，反斜杠
```

字符字面量**类型永远是 `u8`**，值是对应的 ASCII / UTF-8 字节（因此值域是 0..=127 的 ASCII；更高位要用 `\xNN` 转义，0..=0xFF 都允许）。

不支持多字节 Unicode 字符字面量——Aperio 的字符级类型就是字节。要写多字节序列，用字符串字面量。

## 字符串字面量

```rust
"Hello"         // u8[5]，无 \0
c"Hello"        // u8[6]，末尾追加 \0
```

字符串字面量是一个**数组常量**——类型是 `u8[N]`，`N` 是字节数（`c"..."` 的 `N` 包含末尾的 `\0`）。详见 [19. 字符串字面量](./19_strings.md)。

用在数据段定义里：

```rust
val MSG: u8[] = c"Hello, Aperio!\n"
```

用在宏参数里：

```rust
io::println!("value = {d}", r1)
debug::assert!(cond, "must be non-zero")
```

## 布尔字面量

```rust
true            // bool
false           // bool
```

只出现在需要 `bool` 的位置：`if` 条件、`std/core::select` 的第一参数、`#[cfg(...)]` 条件。布尔字面量和整数字面量**不能互换**：

```rust
if (1 > 0) goto(@x)            // OK，比较产出 bool
if (1) goto(@x)                // 编译错：需要 bool，得到 i64

r1 = true                      // 编译错：r1 是整数寄存器，承载 bool 要显式 as
r1 = true as u8                // OK：1
```

## 地址字面量 `&<symbol>`

取地址运算符产出的值也可以看作一个字面量类的表达式——它的类型是**指向该符号类型的指针**：

```rust
val BUF: u8[256]
r1 = &BUF                      // r1 当前类型 = *u8（数组衰减）

type Point = struct { x: i32, y: i32 }
val P: Point = { x: 0, y: 0 }
r2 = &P                        // r2 当前类型 = *Point
```

详见 [09. 取地址](./09_address_of.md)。

## 总结表

| 字面量                 | 默认类型 | 可后缀 | 可上下文收缩 |
|------------------------|----------|--------|--------------|
| `42`                   | `i64`    | `i*`/`u*` | 是         |
| `0xFF`                 | `i64`    | `i*`/`u*` | 是         |
| `3.14`                 | `f64`    | `f32`/`f64` | 是       |
| `'A'`                  | `u8`     | 否     | 否           |
| `"abc"`                | `u8[3]`  | 否     | 否           |
| `c"abc"`               | `u8[4]`  | 否     | 否           |
| `true` / `false`       | `bool`   | 否     | 否           |
| `&SYMBOL`              | `*T`     | 否     | 否           |

"可上下文收缩"表示编译器会按上下文把字面量类型收缩到比默认更窄的整数/浮点类型（只要值在范围内）。
