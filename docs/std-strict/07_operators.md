# 07. 运算符

Aperio 使用 C 风格的运算符语法，而不是汇编助记符。`r1 = r2 + r3` 比 `add r1, r2, r3` 更符合现代开发者的肌肉记忆。

## 算术运算符

| 运算符 | 含义           | 整数等价 | 浮点等价 |
|--------|----------------|----------|----------|
| `+`    | 加法           | `add`    | `fadd`   |
| `-`    | 减法 / 取负    | `sub` / `neg` | `fsub` / `fneg` |
| `*`    | 乘法           | `imul` / `mul` | `fmul` |
| `/`    | 除法           | `idiv` / `div` | `fdiv` |
| `%`    | 取模（仅整数） | `idiv` / `div` | —      |

```rust
r1 = r2 + r3
r1 = r2 - r3
r1 = r2 * r3
r1 = r2 / r3
r1 = r2 % r3
r1 = -r2          // 取负（一元减号）

f1 = f2 + f3
f1 = f2 - f3
f1 = f2 * f3
f1 = f2 / f3
f1 = -f2
```

**操作数类型必须一致**（同类别 + 同位宽）；整数和浮点不能直接混用。两端都是 `r*`（整数/指针）或都是 `f*`（浮点）。

### 有符号 vs 无符号（整数）

乘法、除法、取模、右移的**操作本身**区分有符号 / 无符号。编译器按两个操作数的**当前类型**决定：

- 两侧都是 `i*` → 有符号版本（`idiv`、`sar`）
- 两侧都是 `u*` → 无符号版本（`div`、`shr`）
- 两侧类型不一致 → 编译错

详见 [13. ABI 的寄存器类型流](./13_abi.md#寄存器类型流dreg)。

### 浮点运算的语义

浮点运算严格按 IEEE 754：

- `0.0 / 0.0` → NaN
- `1.0 / 0.0` → +∞，`-1.0 / 0.0` → -∞
- 溢出 → ±∞
- 任何带 NaN 的比较（除了 `!=`）→ `false`

浮点运算**不触发陷阱**——除零、溢出都按规范走。想要 `int / 0` 式的陷阱行为请用整数除法。

浮点 `%` 不支持——需要的话走 C 库 `fmod`（通过 `extern fn`）。

### 同时需要商和余数

写成两条独立语句，朴素 codegen 会**发两条除法指令**：

```rust
r1 = r4 / r5             // 一条 DIV
r2 = r4 % r5             // 又一条 DIV
```

硬件（x86 `DIV` / `IDIV`）其实一条指令就能同时产出两者。想拿到语言层面的保证，用**多目标赋值**：

```rust
(r1, r2) = r4 / r5       // r1 = 商，r2 = 余；保证只发一条除法
```

- 左侧必须恰好两个寄存器，用括号括起来（和多返回值函数的解构语法一致）
- 顺序固定：第一个是商，第二个是余
- 语义和 `/` / `%` 完全一致——有符号 `i*` 时向零取整、余数与被除数同号；无符号 `u*` 时自然数除法
- UB 条件也一样：除零、有符号的 `T::MIN / -1`（见 [38. 未定义行为](./38_undefined_behavior.md)）

两个目标寄存器必须是**不同的**寄存器；类型上它们分别承接"商"和"余"（都是被除数的类型）：

```rust
r1: i64 = 17i64
r2: i64 = 5i64
(r3, r4) = r1 / r2       // r3: i64 = 3，r4: i64 = 2

(r3, r3) = r1 / r2       // 编译错：两个目标必须不同
```

这个多目标写法是 `/` 运算符**专有**的——`+` / `-` / `*` / `%` / 位运算都不支持（没有硬件一条指令同时产两个值的语义）。浮点也不支持（没有余数概念）。

### 字面量类型推导

无后缀字面量在算术表达式里**跟随另一个操作数的当前类型**：

```rust
r1: u8
r2 = r1 + 1                  // 1 被推导为 u8
```

详见 [06. 字面量](./06_literals.md)。

## 位运算符（仅整数）

| 运算符 | 含义       |
|--------|------------|
| `<<`   | 左移       |
| `>>`   | 右移       |
| `&`    | 按位与     |
| `\|`   | 按位或     |
| `^`    | 按位异或   |
| `~`    | 按位取反   |

```rust
r1 = r2 << 3
r1 = r2 >> r3
r1 = r2 & 0xFF
r1 = r2 | 0x80
r1 = r2 ^ r3
r1 = ~r2
```

浮点寄存器不支持位运算——想操作浮点的位表示，先走 `conv::bitcast_f64_to_i64` 拿到整数寄存器。

### 关于右移

- 有符号 `i*` 右移：高位填符号位（算术右移，`sar`）
- 无符号 `u*` 右移：高位填 0（逻辑右移，`shr`）

### 位移量的约束

位移量 ≥ 位宽是 UB（详见 [38. 未定义行为](./38_undefined_behavior.md)）。要确定性的循环移位请用 `std/bits::rotate_left_*` / `rotate_right_*`。

## 比较运算符

比较运算符产出 **`bool`**。它们可以独立参与赋值，也可以直接用作 `if` 条件：

| 运算符 | 含义     |
|--------|----------|
| `==`   | 等于     |
| `!=`   | 不等于   |
| `<`    | 小于     |
| `>`    | 大于     |
| `<=`   | 小于等于 |
| `>=`   | 大于等于 |

```rust
if (r1 == r2) goto(@equal)
if (r1 > 0)   goto(@positive)

r3: bool = r1 < r2                    // 结果当 bool 用（配合 select）
r0 = core::select(r3, r4, r5)
```

比较两侧的类型必须一致；浮点比较按 IEEE 754。

## 取地址运算符

`&` 在**一元位置**时表示取地址，在**二元位置**时表示按位与：

```rust
r1 = &G_BUFFER     // 一元 &：取地址，r1: *u8
r2 = r3 & 0xFF     // 二元 &：按位与
```

详见 [09. 取地址](./09_address_of.md)。

## `as` 转换

`<expr> as <type>` 做**同类别**的位宽截断/扩展（整数内）或位重解释。允许的方向有限：

| 方向                       | 含义                               |
|----------------------------|------------------------------------|
| `T as U`（同类整数）        | 整数之间宽度调整（按目标符号性扩展/截断）|
| `*T as *U`                 | 指针类型改写（同位宽重解释）        |
| `*T as u64` / `u64 as *T`  | 指针 ↔ 地址                        |
| `*T as *void`              | 任意类型指针 → 不透明指针           |
| `bool as u8`               | `true` → 1，`false` → 0            |

**不允许**的方向要走 `std/convert`：

- 浮点 ↔ 整数
- 浮点 ↔ 浮点（`f32` ↔ `f64`）
- 位重解释（整数的位 → 浮点）

```rust
r1 = r2 as u8                        // OK，整数内截断
p  = addr as *i32                    // OK，地址转指针
f1 = r1 as f32                       // 编译错：走 conv::i32_to_f32
```

## 运算符优先级

从高到低：

1. 一元运算符：`-`（取负）、`~`（按位取反）、`&`（取地址）、`as`
2. `*` `/` `%`
3. `+` `-`
4. `<<` `>>`
5. `&`（按位与）
6. `^`
7. `|`
8. 比较：`==` `!=` `<` `>` `<=` `>=`

圆括号 `()` 强制指定求值顺序：

```rust
r1 = (r2 + r3) * r4
r1 = r2 & (0xFF << 8)
```

## 不支持的运算符

- 逻辑运算符 `&&`、`||`、`!`（用显式分支或 `select` 替代）
- 复合赋值 `+=`、`-=`、`*=`（写 `r1 = r1 + r2` 即可，编译器会优化）
- 自增自减 `++`、`--`
- 三元运算符 `? :`（用 `std/core::select` 或分支替代）
- 指针解引用 `*p`（用 `mem.T[p]` 语法）
- 指针算术 `p + n`（用 `std/ptr::byte_offset` / `offset<T>`）
