# 09. 取地址运算符

`&` 在一元位置时用于获取一个符号的内存地址。它是 Aperio 里唯一用来把"名字"转成"指针"的桥梁。

## 语法

```
& <name>
```

`<name>` 是一个已声明的符号，可以是：

- 全局常量（`val`）
- 可变全局变量（`var`）
- 函数名

## 结果的类型

取地址的结果是一个**带类型的指针**。具体的类型取决于被取地址的对象：

| 被取对象            | `&` 的结果类型            | 说明                     |
|---------------------|---------------------------|--------------------------|
| `val X: i32`        | `*i32`                    | 标量 → 指向标量的指针    |
| `val P: Point`      | `*Point`                  | 结构体 → 指向结构体的指针 |
| `val BUF: u8[256]`  | `*u8`                     | 数组 **衰减** 为元素指针 |
| 函数 `f`            | `fn(...) -> ...` 指针     | 带完整签名的函数指针     |

### 数组衰减

`&ARRAY_NAME` 的结果是**指向元素类型的指针**，长度信息丢失——和 C 的数组衰减一致：

```rust
val BUF: u8[256] = [0; 256]

r1 = &BUF                     // r1 的当前类型 = *u8（不是 "u8[256] 的指针"）
r2 = BUF.length               // 长度通过编译期常量另外获取，= 256u64
```

如果你需要保留"指针 + 长度"的一对信息，自己传两个寄存器：

```rust
fn write_all(r1: *u8, r2: u64) -> (r0: i64) { ... }

r0 = write_all(r1 = &BUF, r2 = BUF.length)
```

**不支持** `*[T; N]` 这种带长度的指针类型——Std-Strict 刻意保持指针类型的窄。

### 结构体不衰减

结构体不衰减，`&P` 就是 `*Point`：

```rust
type Point = struct { x: i32, y: i32 }
val P: Point = { x: 0, y: 0 }

r1 = &P                       // r1 的当前类型 = *Point
r2 = mem.i32[r1 + Point.x]    // 按字段偏移读
```

详见 [20. 结构体](./20_structs.md)。

### 函数指针

对函数取地址产生一个**带签名**的函数指针，可以保存在符合签名的 `type` 别名里：

```rust
fn add(r1: i64, r2: i64) -> (r0: i64) {
    r0 = r1 + r2
}

type BinaryOp = fn(r1: i64, r2: i64) -> (r0: i64)

pub fn demo() -> (r0: i64) uses r9 {
    r9 = &add                            // r9 当前类型 = fn(r1:i64, r2:i64) -> (r0:i64)
    r9(r1 = 10, r2 = 20)                 // 通过 r9 间接调用（r0 = 30）
}
```

详见 [16. 函数指针](./16_function_pointers.md)。

## 不能取的东西

以下情况**不能**使用 `&`：

- 寄存器：`&r1` 非法，寄存器本身不是内存对象
- 编译期常量：`&MAX_SIZE` 非法，`const` 是纯编译期数值，运行时不占内存
- 字面量：`&42` 非法
- 结构体字段：`&Point.x` 非法（字段偏移是编译期数值，不是地址）

如果需要临时的可寻址内存，用 [`std/stack::alloc<T>`](./32_std_stack.md)。

## 与按位与 `&` 的区分

二元位置的 `&` 是按位与运算符，语法上和取地址完全不冲突：

```rust
r1 = &MSG              // 一元 & —— 取地址（*u8）
r2 = r3 & 0xFF         // 二元 & —— 按位与
r4 = (&MSG as u64) & 0xFFF
                       // 先取地址再按位与，注意要先 as u64
```

按位与只接受整数（`r*`）——想对指针做按位运算必须先 `as u64`。
