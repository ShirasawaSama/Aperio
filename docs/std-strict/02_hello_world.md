# 02. Hello World

在深入任何语法细节之前，我们先看一个能跑起来的完整程序，对 Std-Strict 的样貌有一个直观印象。

## 代码

```rust
import "std/io" as io

export pub fn main() -> (r0: i32) {
    io::println!("Hello, Aperio!")
    r0 = 0i32
}
```

这就是全部。真的。

## 逐行解读

### `import "std/io" as io`

把标准库的 `std/io` 模块引进来，起个本地别名 `io`。Aperio 没有 prelude——**所有**想用的东西都得显式 import。`io::println!` 就是通过这个别名访问的。模块系统的细节见 [21. 模块](./21_modules.md)。

### `export pub fn main() -> (r0: i32)`

定义可执行程序的入口函数：

- `export` 让链接器看见这个符号
- `pub` 对其他模块可见
- `()` 不接受参数（想接收 `argc` / `argv` 见 [36. 程序入口与启动](./36_startup.md)）
- `-> (r0: i32)` 返回值放在 `r0`，类型是 `i32`；这个值会被编译器注入的 `_start` 作为进程退出码

### `io::println!("Hello, Aperio!")`

打印一行到 stdout，**自动加换行**。`!` 后缀表示这是个**宏**，不是普通函数——因为 `println!` 要在编译期解析格式串并按参数类型选择打印路径。宏系统见 [23. 宏系统](./23_macros.md)，`std/io` API 见 [26. std/io](./26_std_io.md)。

### `r0 = 0i32`

把返回值设为 0。字面量写成 `0i32` 是因为 `r0` 在签名里声明为 `i32`，Aperio 的字面量类型必须精确匹配（见 [06. 字面量](./06_literals.md)）。

函数没写 `return`——**尾部的 `return` 可省略**。

## 关键观察

几个值得你现在就注意的点：

1. **没有 `add`、`mov` 这类助记符**。赋值、算术、比较都用 C 风格的运算符
2. **`!` 后缀的是宏**，普通函数（如 `io::print`，如果存在）不带 `!`
3. **字面量带类型后缀**是常态。`0` 默认是 `i64`；想写 `i32` 就写 `0i32`
4. **所有东西都要 import**。没有隐藏的全局
5. **`uses` 在这里省略了**，因为本函数除了参数 / 返回值没有动过其他寄存器。等我们进入需要临时寄存器的例子时再引入

## 一个用到临时寄存器的版本

让 `main` 在调用 `println!` 前先做一点计算：

```rust
import "std/io" as io

pub fn square(r1: i32) -> (r0: i32) {
    r0 = r1 * r1
}

export pub fn main() -> (r0: i32) uses r1 {
    r1 = square(r1 = 7i32)                    // r1: i32 = 49
    io::println!("7 squared = {}", r1)
    r0 = 0i32
}
```

这里多出来三件事：

1. **命名参数调用**：`square(r1 = 7i32)`。普通函数调用**必须**用命名参数形式——这比"先铺寄存器再调用"的纯汇编风格不容易出错
2. **`uses r1`**：声明本函数除了参数 / 返回值还会动 `r1`。编译器会做全局的寄存器冲突检查
3. **`{}` 通用占位符**：格式串里的 `{}` 根据参数类型自动选择打印形式。见 [26. std/io](./26_std_io.md)

## 编译

```bash
aperio build hello.ap -o hello
./hello
# Hello, Aperio!
```

接下来的章节会把上面涉及的每个概念单独展开讲。如果你已经被 `r1` / `r2` 这种编号命名冒犯到了，别急——Std-Strict 提供[寄存器别名](./05_registers.md#寄存器别名)机制，可以给每个槽起人类可读的名字。本章故意用裸 `r*` 写法以保持第一印象"这是底层 IR"，真实工程代码里别名版更常见。
