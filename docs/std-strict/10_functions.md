# 10. 函数

函数是 Std-Strict 中唯一的代码组织单位。本章讲函数的**定义**、**调用**和**返回**。关于具体的寄存器分配规则，请参考 [11. 调用约定 ABI](./11_abi.md)。

## 定义函数

函数定义的完整语法：

```
[export] [pub] [inline] fn <name>(<params>) -> <return> [uses <scratch>] {
    <body>
}
```

一个最小的例子：

```text
fn add(r1, r2) -> r0 {
    r0 = r1 + r2
}
```

### 修饰符

| 修饰符   | 含义                                                   |
|----------|--------------------------------------------------------|
| `export` | 导出符号，让链接器和其他模块能看见                     |
| `pub`    | 对本模块外可见（不加则仅模块内可见）                   |
| `inline` | 在调用处展开，而非产生真正的 call 指令，详见第 12 章   |

`export` 和 `pub` 不是同一件事：`pub` 控制 Aperio 模块系统层面的可见性；`export` 控制底层链接器层面的符号可见性。它们可以独立组合。

### 参数列表

参数列表声明**传入参数使用的寄存器**，而不是参数的类型和名字。参数按声明顺序对应到调用方提前设置好的寄存器。

```text
fn distance(r1, r2, r3, r4) -> r0 uses (r5, r6) {
    // r1 = x1, r2 = y1, r3 = x2, r4 = y2
    // 约定如此，但语言不强制
}
```

Std-Strict 不跟踪参数的类型——你自己记住每个寄存器里放的是什么。如果需要类型检查，可以使用 Loose 模式或者 FFI 声明。

### 返回值

`-> r0` 声明返回值会被放在 `r0` 中。返回寄存器可以是任意 `r0` ~ `r999`——不必是 `r0`，但按惯例通常用 `r0`。

#### 多返回值

一个函数可以返回多个寄存器。**多返回值必须用括号包围**：

```text
fn divmod(r1, r2) -> (r0, r3) {
    r0 = r1 / r2       // 商
    r3 = r1 % r2       // 余
}
```

单返回值括号可写可不写（`-> r0` 与 `-> (r0)` 等价）；两个及以上**必须**写括号。

调用多返回值函数后，所有被返回的寄存器都是"活的"，调用方需用解构式接收（见后文"调用多返回值函数"）。

### uses 子句：声明临时寄存器

`uses (...)` 列出函数使用的**临时寄存器**（scratch registers）——那些既不是参数也不是返回值，但在函数体内会被写入的寄存器。

```text
fn distance(r1, r2, r3, r4) -> r0 uses (r5, r6) {
    r5 = r3 - r1       // dx
    r6 = r4 - r2       // dy
    r0 = r5 * r5 + r6 * r6
}
```

#### 语法细节

- **0 个临时寄存器**：省略整个 `uses` 子句
- **1 个临时寄存器**：括号可选，`uses r5` 与 `uses (r5)` 等价
- **2 个及以上**：**必须**用括号，`uses (r5, r6)`
- **连续区间**：`uses r3..=r10` 表示 `r3, r4, ..., r10`；`uses r3..r10` 表示 `r3, r4, ..., r9`（排除右端）
- **混合**：`uses (r3..=r5, r7, r9..=r10)`

```text
fn simple(r1) -> r0 {                              // 0 个，省略
    r0 = r1 + 1
}

fn need_one(r1) -> r0 uses r3 {                    // 1 个，免括号
    r3 = r1 * 2
    r0 = r3 + 1
}

fn need_several(r1, r2) -> r0 uses (r3, r4, r5) {  // 多个，带括号
    r3 = r1 + r2
    r4 = r3 * 2
    r5 = r4 - 1
    r0 = r5
}

fn big_scratch(r1) -> r0 uses r3..=r12 {           // 连续区间
    // 10 个临时寄存器
}
```

#### 为什么需要显式声明

`uses` 是**函数契约的一部分**，和参数、返回值一起构成完整的 ABI 声明。它让调用方一眼看出：

- 调用前需要铺哪些参数寄存器
- 调用后哪些寄存器有返回值
- 调用后哪些寄存器的值被破坏了

显式声明 `uses` 避免了"函数改动了未声明的寄存器"这类隐藏 bug——编译器会静态检查函数体里的所有寄存器写入，发现未声明的写入立即报错：

```text
fn bad(r1) -> r0 uses r3 {
    r3 = r1 + 1
    r5 = r3 * 2      // 编译错误：r5 未在签名中声明
    r0 = r5
}
```

## 调用函数

在 Std-Strict 模式下，函数调用**必须使用命名参数**：

```text
<fn-name>(<reg1> = <expr1>, <reg2> = <expr2>, ...)
```

其中 `<regN>` 必须是被调用函数签名里声明的参数寄存器名字。例如：

```text
fn add(r1, r2) -> r0 {
    r0 = r1 + r2
}

pub fn main() -> r0 {
    add(r1 = 10, r2 = 20)      // 调用后 r0 = 30
}
```

命名形式是**强制**的——让每次调用在源码里都自我说明："这个调用把哪些值放进了哪些参数寄存器"。想写纯汇编式"铺寄存器 + 空括号"，请使用 Native-Strict 模式。

### 赋值项

`<reg> = <expr>` 中：

- 左边必须是**参数寄存器**名字（必须出现在被调用函数签名的参数列表里）
- 右边是任意能作为表达式右值的内容：立即数、寄存器、内存读取、`&symbol` 等

```text
my_func(
    r1 = 10,
    r2 = r5 + 1,
    r3 = mem.i32[r7 + 4],
    r4 = &BUFFER,
)
```

### 必须写全所有参数

所有参数都必须在调用处显式给值，不能省略。想"沿用当前值"就写 `rN = rN`：

```text
r1 = compute_first()
add(r1 = r1, r2 = 20)        // 显式写 r1 = r1 表示"沿用 r1"

// 非法：
// add(r2 = 20)              // 错误：r1 未提供
```

这个规则避免了"我忘了 r1 当前是什么值"这种隐藏问题。

### 求值顺序

编译器保证赋值按**源码书写顺序**从上到下求值。如果右边表达式与目标寄存器有相互依赖（比如 `fn(r1 = r2, r2 = r1)`），编译器会自动引入临时寄存器来解环——这和手动交换变量时需要 `tmp = a; a = b; b = tmp` 的底层逻辑是一样的，只是编译器替你处理。

## 调用多返回值函数

对于签名为 `-> (r0, r3)` 的函数，调用方**必须**用解构式接收返回值：

```text
(r0, r3) = divmod(r1 = 17, r2 = 5)
// 现在 r0 = 3, r3 = 2
```

左边括号里写出签名中声明的**所有**返回寄存器，顺序必须与签名一致。允许用 `_` 丢弃某个返回值：

```text
(r0, _) = divmod(r1 = 17, r2 = 5)    // 只关心商
(_, r3) = divmod(r1 = 17, r2 = 5)    // 只关心余
```

不允许少写某个返回值——`r0 = divmod(...)` 对多返回值函数是**非法**的。

### 单返回值的接收

单返回值函数允许两种写法，等价：

```text
add(r1 = 10, r2 = 20)             // 不显式接收，结果自然在 r0
r0 = add(r1 = 10, r2 = 20)        // 显式接收
```

前一种更常见，因为返回值的落脚点已经在签名里写明了，没必要重复。

## 间接调用

如果你有一个寄存器里存着函数地址，语法与普通调用一致，只是函数名换成了寄存器：

```text
r9 = &my_func
r9(r1 = 10, r2 = 20)
```

多返回值的间接调用：

```text
r9 = &divmod
(r0, r3) = r9(r1 = 17, r2 = 5)
```

对应 x86 的 `call rax` 或 Wasm 的 `call_indirect`。间接调用必须符合被调用函数的 ABI，这一点编译器无法静态验证——使用者需要自己保证。

## 返回

### 显式 return

在函数中间提前返回时使用 `return`：

```text
fn abs(r1) -> r0 {
    if (r1 >= 0) goto(@positive)
    r0 = -r1
    return                  // 必需：提前返回

@positive:
    r0 = r1
}
```

`return` 本身**不带表达式**。返回值必须在执行 `return` 之前放入签名声明的返回寄存器中：

```text
fn square(r1) -> r0 {
    r0 = r1 * r1
    return
}
```

### 尾部 return 可省略

函数体的最后一条语句如果是"隐式返回"（即控制流自然走到函数结尾），可以省略 `return`：

```text
fn square(r1) -> r0 {
    r0 = r1 * r1        // 正常地把结果放入 r0
}                       // 函数结束时自动 return
```

这只适用于函数末尾。中间的分支返回仍然需要显式写 `return`。

### 多返回值的 return

多返回值函数在 `return`（或函数末尾隐式返回）前，**所有**签名声明的返回寄存器都必须被赋值过。编译器会对此做静态检查：

```text
fn divmod(r1, r2) -> (r0, r3) {
    r0 = r1 / r2
    r3 = r1 % r2
}                          // OK

fn bad(r1, r2) -> (r0, r3) {
    r0 = r1 / r2
    // 编译错误：r3 在函数返回前未被赋值
}
```

## 可见性小结

| 修饰组合              | 可见范围                             |
|-----------------------|--------------------------------------|
| `fn foo`              | 仅本文件内（模块私有）               |
| `pub fn foo`          | 本模块的所有文件 + 其他模块可 `import` |
| `export fn foo`       | 仅本文件内，但符号在二进制中可被链接 |
| `export pub fn foo`   | 全局可见，符号可被链接               |
