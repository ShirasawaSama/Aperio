# 04. 数据类型

Aperio 提供一组精确到位宽的标量类型。类型系统的职责只有两个：**告诉编译器一块内存有多大**，以及**是否是有符号的**。不存在高级语言里那些复合类型（数组类型、引用类型、泛型）。

## 整数类型

| 类型   | 位宽 | 有符号 | 取值范围                          |
|--------|------|--------|-----------------------------------|
| `i8`   | 8    | 是     | -128 ~ 127                        |
| `i16`  | 16   | 是     | -32768 ~ 32767                    |
| `i32`  | 32   | 是     | -2^31 ~ 2^31-1                    |
| `i64`  | 64   | 是     | -2^63 ~ 2^63-1                    |
| `u8`   | 8    | 否     | 0 ~ 255                           |
| `u16`  | 16   | 否     | 0 ~ 65535                         |
| `u32`  | 32   | 否     | 0 ~ 2^32-1                        |
| `u64`  | 64   | 否     | 0 ~ 2^64-1                        |

## 浮点类型

| 类型   | 位宽 | 标准        |
|--------|------|-------------|
| `f32`  | 32   | IEEE 754 单精度 |
| `f64`  | 64   | IEEE 754 双精度 |

## 类型在哪里出现

你**不会**在寄存器上看到类型，因为 Std-Strict 的寄存器是无类型的——它们只是存储 64 位值的容器，怎么解释这些位取决于对它们执行的操作。

类型只出现在以下三个地方：

### 1. 内存访问指令上

```text
r2 = mem.i32[r1]       // 按 i32 读取（4 字节，有符号扩展到 64 位）
r3 = mem.u8[r1 + 4]    // 按 u8 读取（1 字节，零扩展到 64 位）
mem.f64[r1 + 8] = r4   // 按 f64 写入（8 字节）
```

### 2. 数据段定义上

```text
const MAX: i32 = 1024
val BUF: u8[256]
var COUNTER: i64[1] = [0]
```

### 3. 函数参数的 FFI 声明上

```text
extern fn printf(fmt: u64, val: i32) -> i32
```

## 符号扩展与零扩展

由于所有寄存器都是 64 位宽，从内存加载窄类型时必须明确扩展策略：

- 从 `i*` 类型加载 → **符号扩展**（高位填充符号位）
- 从 `u*` 类型加载 → **零扩展**（高位填充 0）

这和 x86-64 的 `movsx` / `movzx` 指令对应。写入内存时则直接截断到目标位宽。

```text
// 假设 mem[r1] 的字节是 0xFF
r2 = mem.i8[r1]    // r2 = 0xFFFFFFFFFFFFFFFF （-1，符号扩展）
r3 = mem.u8[r1]    // r3 = 0x00000000000000FF （255，零扩展）
```

## 浮点与整数的关系

Std-Strict **不会**自动在整数和浮点之间转换。如果你需要把 `i32` 转成 `f32`，需要使用显式的转换指令（后续章节会介绍）。直接混用会被编译器拒绝：

```text
r1 = mem.i32[addr1]
r2 = mem.f32[addr2]
r3 = r1 + r2           // 错误：整数和浮点不能直接相加
```
