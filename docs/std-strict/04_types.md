# 04. 数据类型

Aperio 提供一组精确到位宽的标量类型和一族指针类型。类型系统的职责是：**告诉编译器一块内存有多大**、**是否有符号**、以及**一个指针指向什么**。除此之外没有高级语言里那些复合类型（数组类型、引用类型、用户定义泛型）。

## 整数类型

| 类型   | 位宽 | 有符号 | 取值范围                          |
|--------|------|--------|-----------------------------------|
| `i8`   | 8    | 是     | -128 ~ 127                        |
| `i16`  | 16   | 是     | -32768 ~ 32767                    |
| `i32`  | 32   | 是     | -2³¹ ~ 2³¹-1                      |
| `i64`  | 64   | 是     | -2⁶³ ~ 2⁶³-1                      |
| `u8`   | 8    | 否     | 0 ~ 255                           |
| `u16`  | 16   | 否     | 0 ~ 65535                         |
| `u32`  | 32   | 否     | 0 ~ 2³²-1                         |
| `u64`  | 64   | 否     | 0 ~ 2⁶⁴-1                         |

## 浮点类型

| 类型   | 位宽 | 标准              |
|--------|------|-------------------|
| `f32`  | 32   | IEEE 754 单精度   |
| `f64`  | 64   | IEEE 754 双精度   |

## 布尔类型

| 类型   | 位宽 | 说明                              |
|--------|------|-----------------------------------|
| `bool` | 1（存储占 1 字节）| 只取 `true` / `false`，用于条件和 `std/core::select` 的第一参数 |

比较运算符（`==`、`<` 等）产出的值类型是 `bool`，不是整数。想要当整数用需要走 `std/convert`。

## 指针类型

指针类型有一整族：

| 语法     | 语义                                 |
|----------|--------------------------------------|
| `*T`     | 指向类型 `T` 的对齐指针              |
| `*void`  | 不透明指针，不携带被指类型           |

```rust
*i8      *i16      *i32      *i64
*u8      *u16      *u32      *u64
*f32     *f64      *bool
*Point                                  // 指向用户结构体
*void                                   // 不透明
```

指针值本身是一个地址（在本机通常是 64 位），但类型系统会记住"指的是什么"。这让内存访问、算术偏移、数组衰减都能被类型检查。

**指针不是整数**。`*T` 和 `u64` 之间的转换必须显式 `as`：

```rust
r1 = ptr as u64            // 指针 → 整数地址（OK）
r2 = addr as *i32          // 整数 → 指针（OK，由使用者保证合法）
r3 = ptr + 1               // 编译错：禁止用 + 做指针算术
```

指针算术和指针相关的工具函数集中在 [`std/ptr`](./31_std_ptr.md) 模块。内存访问规则见 [08. 内存访问](./08_memory.md)。

### 没有"数组指针类型"

Aperio **不提供** `*[T; N]` 这种带长度的指针类型。数组的取地址结果会**衰减**为指向元素类型的指针（见 [09. 取地址](./09_address_of.md)）：

```rust
val BUF: u8[256] = [0; 256]
r1 = &BUF                   // r1: *u8，长度信息丢失
r2 = BUF.length             // 长度通过编译期常量另外获取
```

这是刻意的简化——让指针类型保持窄，其他信息靠编译期常量补齐。

## 类型出现在哪些地方

Std-Strict 的类型只会在下面几个边界上出现：

### 1. 函数签名

```rust
fn add(r1: i64, r2: i64) -> (r0: i64)
extern fn printf(fmt: *u8, ...) -> (r0: i32)
#[builtin] pub fn select<T>(cond: bool, a: T, b: T) -> T
```

签名里的类型构成了函数的 ABI 契约。调用点和函数体都按这里面的类型进行检查。详见 [13. 调用约定](./13_abi.md)。

### 2. 内存访问指令

```rust
r2 = mem.i32[r1]       // 按 i32 读（4 字节，符号扩展到 64 位）
r3 = mem.u8[r1 + 4]    // 按 u8 读（1 字节，零扩展到 64 位）
mem.f64[r4] = f1       // 按 f64 写
```

访问类型决定位宽、符号扩展策略和返回值类型。指针的被指类型必须与访问类型兼容，详见 [08. 内存访问](./08_memory.md)。

### 3. 数据段定义

```rust
const MAX: i32 = 1024
val BUF: u8[256]
var COUNTER: i64 = 0
var HANDLER: *u8 = &DEFAULT
```

### 4. 字面量后缀

```rust
42i32          0xFFu8        3.14f64         1.0f32
```

详见 [06. 字面量](./06_literals.md)。

### 5. 函数指针类型别名

```rust
type BinaryOp = fn(r1: i64, r2: i64) -> (r0: i64)
```

详见 [16. 函数指针](./16_function_pointers.md)。

## 符号扩展与零扩展

由于整数寄存器都是 64 位宽，从内存加载窄类型时必须明确扩展策略：

- 从 `i*` 类型加载 → **符号扩展**（高位填充符号位）
- 从 `u*` 类型加载 → **零扩展**（高位填充 0）

这和 x86-64 的 `movsx` / `movzx` 指令对应。写入内存时则直接截断到目标位宽。

```rust
// 假设 mem[r1] 的字节是 0xFF
r2 = mem.i8[r1]    // r2 = 0xFFFFFFFFFFFFFFFF （-1，符号扩展）
r3 = mem.u8[r1]    // r3 = 0x00000000000000FF （255，零扩展）
```

## 没有用户级泛型

用户定义的 `fn` **不支持**泛型参数。想要多个位宽就起多个名字：

```rust
fn swap_i32(r1: *i32, r2: *i32) { ... }
fn swap_i64(r1: *i64, r2: *i64) { ... }
```

唯一的例外是编译器内置的 `#[builtin]` 函数，它们可以声明有限的 `<T>` 类型参数——比如 `std/core::select<T>`、`std/ptr::offset<T>`。这是为了给"对多种位宽对称"的原语一个干净签名，不是给用户的通用泛型机制。详见 [24. 内置库总览](./24_builtins_overview.md)。

## 整数与浮点不自动转换

```rust
r1 = mem.i32[addr1]
f1 = mem.f32[addr2]
r3 = r1 + f1           // 编译错：整数寄存器和浮点寄存器不能混用

f2 = conv::i32_to_f32(r1)
f3 = f2 + f1           // OK
```

转换指令集中在 [`std/convert`](./28_std_convert.md)。
