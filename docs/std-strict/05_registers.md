# 05. 虚拟寄存器

Std-Strict 不存在传统意义上的"变量"。你所有的运行时状态都承载在寄存器中。本章讲清楚寄存器的使用规则。

## 寄存器命名

所有通用寄存器统一命名为 `r0` 到 `r999`，共 1000 个可用名称。

```text
r0
r1
r42
r999
```

这只是**命名约定**，不代表目标机器上真的有 1000 个物理寄存器。编译器后端会在代码生成阶段，把这些虚拟名字映射到目标架构的物理寄存器，用不完的部分会溢出（spill）到栈上的临时槽。

## 寄存器是无类型的

寄存器本身不携带类型信息，它们就是 64 位的存储槽。同一个寄存器可以在不同时刻存放不同类型的值：

```text
r1 = 42               // 现在 r1 存的是整数
r1 = mem.f32[r5]      // 现在 r1 存的是浮点数的位表示
r1 = &G_TABLE         // 现在 r1 存的是地址
```

类型的解释权完全取决于你对它执行的操作。

## 位宽

所有寄存器统一为 64 位宽。即使你只用来存一个 `u8`，高位也会被零扩展或符号扩展（取决于加载指令）。详见 [04. 数据类型](./04_types.md)。

## 赋值

最基础的寄存器操作是赋值：

```text
r1 = 42           // 立即数
r2 = r1           // 寄存器到寄存器（相当于 mov）
r3 = r1 + r2      // 表达式求值
r4 = &G_TABLE     // 地址
r5 = mem.i32[r4]  // 内存读取
```

赋值目标（左值）必须是寄存器或内存位置，不能是立即数或常量。

## 调用约定对寄存器的约束

虽然寄存器在语法上都是平等的，但在跨函数调用时，它们的语义受 [调用约定 (ABI)](./11_abi.md) 约束：

- **参数寄存器**：调用者负责在调用前设置
- **返回值寄存器**：被调用者负责在返回前设置
- **Clobbered 寄存器**：被调用者可以自由使用，调用者不应假定它们在调用后还保留原值
- **Callee-saved 寄存器**：被调用者如果要使用，必须自己保存和恢复

这些规则由每个函数的签名显式声明，而不是隐藏在语言规范里。

## 特殊寄存器

Std-Strict 故意**不暴露**以下寄存器：

- 栈指针 `sp`
- 帧指针 `bp`
- 指令指针 `pc`
- 标志位寄存器 `flags`

栈的使用通过 [`stack.alloc`](./17_stack.md) 完成，控制流通过 [`goto` 和条件跳转](./09_control_flow.md) 完成，这样做的目的是让同一份 `.ap` 代码可以无差别地编译到 x86、ARM 以及 WebAssembly（后者根本没有硬件栈指针）。
