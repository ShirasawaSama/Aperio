# 12. 函数

函数是 Std-Strict 中唯一的代码组织单位。本章讲函数的**定义**、**调用**和**返回**。关于具体的寄存器约束、`uses` 契约、以及类型流规则，详见 [13. 调用约定](./13_abi.md)。

## 定义函数

完整语法：

```
[<attrs>] [export] [pub] fn <name>(<params>) -> <return> [uses <scratch>] {
    <body>
}
```

一个最小的例子：

```rust
fn add(r1: i64, r2: i64) -> (r0: i64) {
    r0 = r1 + r2
}
```

### 修饰符

| 修饰符   | 含义                                                   |
|----------|--------------------------------------------------------|
| `export` | 导出符号，让链接器和其他模块能看见                     |
| `pub`    | 对本模块外可见（不加则仅模块内可见）                   |

`export` 和 `pub` 不是同一件事：`pub` 控制 Aperio 模块系统层面的可见性；`export` 控制底层链接器层面的符号可见性。它们可以独立组合，末尾小结给了对照表。

`inline`、`#[cold]`、`#[noreturn]` 等以**属性**形式给出——见 [14. 函数属性](./14_function_attrs.md) 和 [15. 内联函数](./15_inline.md)。

### 参数列表

每个参数写成 `<寄存器>: <类型>` 的形式：

```rust
fn distance(r1: i64, r2: i64, r3: i64, r4: i64) -> (r0: i64) uses (r5, r6) {
    // 入口：r1 r2 r3 r4 的当前类型 = i64
}
```

**类型是签名契约的一部分**。调用点按这里声明的类型检查实参；函数体把参数寄存器的入口当前类型设为声明的类型。详见 [13. ABI 的寄存器类型流](./13_abi.md#寄存器类型流dreg)。

参数寄存器可以是 `r*`、`f*` 混用：

```rust
fn scale(r1: i64, f1: f32) -> (f0: f32) {
    f0 = conv::i32_to_f32(r1 as i32) * f1
}
```

#### 用寄存器别名命名参数

每个参数前面可以用 `@` 给槽起个可读的名字——`name @ slot: type`：

```rust
fn distance(
    x1 @ r1: i64, y1 @ r2: i64,
    x2 @ r3: i64, y2 @ r4: i64,
) -> (dist2 @ r0: i64) uses (dx @ r5, dy @ r6) {
    dx = x2 - x1
    dy = y2 - y1
    dist2 = dx * dx + dy * dy
}
```

一旦在签名里给 `r1` 起了别名 `x1`，函数体内**必须**用 `x1`——写 `r1` 是编译错。调用端也必须用别名作为参数名：

```rust
distance(x1 = 0i64, y1 = 0i64, x2 = 3i64, y2 = 4i64)   // OK
distance(r1 = 0i64, r2 = 0i64, r3 = 3i64, r4 = 4i64)   // 编译错：签名里叫 x1/y1/...
```

别名的完整规则（作用域覆盖、严格替换、文件级 / 函数体级声明、与函数指针类型兼容性的关系）见 [05. 虚拟寄存器 — 寄存器别名](./05_registers.md#寄存器别名)。

### 返回值

`-> (r0: i64)` 声明返回值会被放在 `r0` 里，类型是 `i64`。返回寄存器可以是任意 `r0`–`r999` 或 `f0`–`f999`——不必是 `r0`，但按惯例通常用 `r0`/`f0`。

#### 多返回值

一个函数可以返回多个寄存器。**多返回值必须用括号包围**：

```rust
fn divmod(r1: i64, r2: i64) -> (r0: i64, r3: i64) {
    r0 = r1 / r2       // 商
    r3 = r1 % r2       // 余
}
```

单返回值的括号可写可不写（`-> (r0: i64)` 和 `-> r0: i64` 等价）；两个及以上**必须**写括号。

调用多返回值函数后，所有被返回的寄存器都是"活的"，调用方需用解构式接收（见后文"调用多返回值函数"）。

### `uses` 子句：声明临时寄存器

`uses (...)` 列出函数使用的**临时寄存器**——既不是参数也不是返回值、但在函数体内会被写入的寄存器。

```rust
fn distance(r1: i64, r2: i64, r3: i64, r4: i64) -> (r0: i64) uses (r5, r6) {
    r5 = r3 - r1                     // dx
    r6 = r4 - r2                     // dy
    r0 = r5 * r5 + r6 * r6
}
```

`uses` 里**只写寄存器名，不写类型**——临时寄存器的类型随体内赋值自由浮动（详见 [13. ABI 的寄存器类型流](./13_abi.md#寄存器类型流dreg)）。

`uses` 列表里的槽也可以用 `@` 起别名——`uses (dx @ r5, dy @ r6)`。绑定之后函数体内只能用别名。

#### 语法细节

- **0 个临时寄存器**：省略整个 `uses` 子句
- **1 个临时寄存器**：括号可选，`uses r5` 与 `uses (r5)` 等价
- **2 个及以上**：**必须**用括号，`uses (r5, r6)`
- **连续区间**：`uses r3..=r10` 表示 `r3, r4, ..., r10`；`uses r3..r10` 表示 `r3, r4, ..., r9`（排除右端）
- **混合**：`uses (r3..=r5, r7, r9..=r10, f0..=f3)`

```rust
fn simple(r1: i64) -> (r0: i64) {
    r0 = r1 + 1
}

fn need_one(r1: i64) -> (r0: i64) uses r3 {
    r3 = r1 * 2
    r0 = r3 + 1
}

fn need_several(r1: i64, r2: i64) -> (r0: i64) uses (r3, r4, r5) {
    r3 = r1 + r2
    r4 = r3 * 2
    r5 = r4 - 1
    r0 = r5
}

fn big_scratch(r1: i64) -> (r0: i64) uses r3..=r12 {
    // 10 个临时寄存器
}
```

#### 为什么需要显式声明

`uses` 是**函数契约的一部分**，和参数、返回值一起构成完整的 ABI 声明。它让调用方一眼看出：

- 调用前需要铺哪些参数寄存器
- 调用后哪些寄存器有返回值
- 调用后哪些寄存器的值被破坏了

显式声明 `uses` 避免了"函数改动了未声明的寄存器"这类隐藏 bug——编译器会静态检查函数体里的所有寄存器写入，发现未声明的写入立即报错。

## 调用函数

在 Std-Strict 里，用户定义的普通函数调用**必须使用命名参数**：

```rust
<fn-name>(<reg1> = <expr1>, <reg2> = <expr2>, ...)
```

其中 `<regN>` 必须是被调用函数签名里声明的参数寄存器名字：

```rust
fn add(r1: i64, r2: i64) -> (r0: i64) {
    r0 = r1 + r2
}

pub fn main() -> (r0: i64) {
    add(r1 = 10, r2 = 20)                    // 调用后 r0 = 30
}
```

命名形式是**强制**的——让每次调用在源码里都自我说明："这个调用把哪些值放进了哪些参数寄存器"。

> **唯一例外**：`#[builtin]` 函数和 `extern fn` 允许**位置参数**或**命名参数**，由调用方按口味选。详见 [24. 内置库总览](./24_builtins_overview.md)。

### 赋值项

`<reg> = <expr>` 中：

- 左边必须是**参数寄存器**名字
- 右边是任意能作为表达式右值的内容：立即数、寄存器、内存读取、`&symbol` 等

```rust
my_func(
    r1 = 10,
    r2 = r5 + 1,
    r3 = mem.i32[r7 + 4],
    r4 = &BUFFER,
)
```

### 实参类型必须与签名匹配

每个实参的当前类型必须与签名参数类型一致，否则编译错：

```rust
fn takes_i64(r1: i64) -> (r0: i64) { ... }

r5 = mem.u8[r1]                                   // r5 当前类型 = u8
takes_i64(r1 = r5)                                // 编译错：u8 ≠ i64
takes_i64(r1 = r5 as i64)                         // OK（as 转换）
takes_i64(r1 = conv::zero_extend_from_u8(r5))     // OK（推荐）
```

### 必须写全所有参数

所有参数都必须在调用处显式给值，不能省略。想"沿用当前值"就写 `rN = rN`：

```rust
add(r1 = r1, r2 = 20)        // 显式写 r1 = r1 表示"沿用 r1"

// 非法：
// add(r2 = 20)              // 错误：r1 未提供
```

### 求值顺序

命名参数按**源码书写顺序**（左→右）求值。实参表达式如果与目标寄存器有相互依赖（比如 `fn(r1 = r2, r2 = r1)`），编译器会自动引入临时寄存器来解环——和手动交换变量时需要 `tmp = a; a = b; b = tmp` 的底层逻辑一致，只是编译器替你处理。

位置参数（仅 `#[builtin]` / `extern fn` 支持）同样左→右求值。

## 调用多返回值函数

签名 `-> (r0: i64, r3: i64)` 的函数，调用方**必须**用解构式接收返回值：

```rust
(r0, r3) = divmod(r1 = 17, r2 = 5)
// 现在 r0 = 3, r3 = 2，类型都是 i64
```

左边括号里写出签名中声明的**所有**返回寄存器，顺序必须与签名一致。允许用 `_` 丢弃某个返回值：

```rust
(r0, _) = divmod(r1 = 17, r2 = 5)    // 只关心商
(_, r3) = divmod(r1 = 17, r2 = 5)    // 只关心余
```

不允许少写——`r0 = divmod(...)` 对多返回值函数是**非法**的。

### 单返回值的接收

两种写法等价：

```rust
add(r1 = 10, r2 = 20)             // 不显式接收，结果自然在 r0
r0 = add(r1 = 10, r2 = 20)        // 显式接收
```

前一种更常见——返回值的落脚点已经在签名里写明了。

## 间接调用

如果寄存器里存着函数地址，语法与普通调用一致，只是函数名换成了寄存器（或表达式）：

```rust
r9 = &my_func
r9(r1 = 10, r2 = 20)

(r0, r3) = r9(r1 = 17, r2 = 5)
```

间接调用的类型安全建立在**函数指针类型**上——详见 [16. 函数指针](./16_function_pointers.md)。

## 返回

### 显式 `return`

在函数中间提前返回时使用 `return`：

```rust
fn abs(r1: i64) -> (r0: i64) {
    if (r1 >= 0) goto(@positive)
    r0 = -r1
    return

@positive:
    r0 = r1
}
```

`return` **不带表达式**。返回值必须在 `return` 之前放入签名声明的返回寄存器中。

### 尾部 `return` 可省略

函数体的最后一条语句如果控制流自然走到结尾，可以省略 `return`：

```rust
fn square(r1: i64) -> (r0: i64) {
    r0 = r1 * r1
}
```

### 多返回值的 `return`

多返回值函数在返回前，**所有**签名声明的返回寄存器都必须被赋值过，且当前类型等于签名类型。编译器会静态检查：

```rust
fn divmod(r1: i64, r2: i64) -> (r0: i64, r3: i64) {
    r0 = r1 / r2
    r3 = r1 % r2
}                          // OK

fn bad(r1: i64, r2: i64) -> (r0: i64, r3: i64) {
    r0 = r1 / r2
    // 编译错：r3 返回前未被赋值
}
```

## 可见性小结

| 修饰组合              | 可见范围                             |
|-----------------------|--------------------------------------|
| `fn foo`              | 仅本文件内（模块私有）               |
| `pub fn foo`          | 本模块的所有文件 + 其他模块可 `import` |
| `export fn foo`       | 仅本文件内，但符号在二进制中可被链接 |
| `export pub fn foo`   | 全局可见，符号可被链接               |
