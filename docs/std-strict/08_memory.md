# 08. 内存访问

内存在 Aperio 的视角里就是一片字节海，访问它需要三个信息：**地址**、**读写宽度**、**符号性**。本章介绍访问内存的语法。

## 基本语法

```
mem.<T>[<address-expression>]
```

- `<T>` 是访问的数据类型：`i8` / `i16` / `i32` / `i64` / `u8` / `u16` / `u32` / `u64` / `f32` / `f64` 或指针类型 `*U`
- `<address-expression>` 是一个产生地址的表达式，必须是**指针类型**或 `u64`

## 读取

`mem.<T>[addr]` 出现在赋值右侧时表示读取：

```rust
r2 = mem.i32[r1]              // 从 r1 读 4 字节有符号整数（符号扩展到 64 位）
r3 = mem.u8[r1]               // 从 r1 读 1 字节无符号整数（零扩展到 64 位）
f1 = mem.f64[r1]              // 从 r1 读 8 字节双精度浮点（走 f1）
p  = mem.*i32[r1]             // 从 r1 读一个 *i32（8 字节指针）
```

读窄于 64 位的类型时，高位按符号性自动扩展（`i*` 符号扩展，`u*` 零扩展）。读浮点进 `f*` 寄存器，读整数/指针进 `r*`。

读取结果的当前类型 = 访问类型 `<T>`。

## 写入

```rust
mem.i32[r1] = r2              // 截断 r2 低 32 位写入
mem.u8[r1]  = r2              // 只写最低字节
mem.f64[r1] = f1              // 写 8 字节浮点
mem.*u8[r1] = r3              // 写一个 *u8 指针
```

写入时超出位宽的部分**直接截断**。写入值的类型必须与访问类型相容（详见"类型规则"一节）。

## 地址表达式

方括号里支持以下几种形式：

### 单个指针 / 地址

```rust
r2 = mem.i32[r1]
```

`r1` 的当前类型必须是指针或 `u64`。

### 指针 + 常量字节偏移

```rust
r2 = mem.i32[r1 + 8]
r2 = mem.i32[r1 - 4]
```

偏移的单位是**字节**，不受指针被指类型影响。这对应 x86 的 `[rax + 8]` 寻址。

### 指针 + 寄存器

```rust
r2 = mem.i32[r1 + r3]
```

### 带缩放因子（数组索引）

```rust
r2 = mem.i32[r1 + r3 * 4]
```

对应 x86 的 `[rax + rbx*4]`，用于按元素宽度索引数组。缩放因子只能是 `1`、`2`、`4` 或 `8`（x86 硬件限制，ARM 后端转换为等价的左移 + 加法）。

### 完整形式

```rust
r2 = mem.i32[r1 + r3 * 4 + 16]
```

对应 x86 的 `[rax + rbx*4 + 16]`。

## 类型规则

`mem.<T>[p]` 要求 `p` 的当前类型满足**下列之一**，否则编译错：

1. `*T`（被指类型恰好匹配）
2. `*void`（不透明指针，由使用者保证合法）
3. `u64`（裸地址，由使用者保证合法）

其他指针类型（如用 `*i32` 访问 `mem.f32[...]`）**必须**显式 `as` 转换：

```rust
p: *i32
f1 = mem.f32[p]                          // 编译错：*i32 ≠ *f32 / *void / u64
f1 = mem.f32[p as *f32]                  // OK
f1 = mem.f32[p as *void]                 // OK
f1 = mem.f32[p as u64]                   // OK（绕过类型系统，自担风险）
```

**目的**：让"读 32 位有符号整数"和"读 32 位浮点"这种位宽相同但语义不同的错配在源码里看得见。想要 punning 就显式走一步 `as` 或 `conv::bitcast_*`。

## 地址空间

默认的 `mem.<T>[...]` 等价于 `mem.global.<T>[...]`，表示全局可寻址的内存（进程的线性地址空间）。为支持 CUDA 等非统一内存模型的目标，预留：

| 前缀         | 含义                       |
|--------------|----------------------------|
| `mem.global` | 全局内存（默认）           |
| `mem.shared` | CUDA 共享内存（block 内）  |
| `mem.local`  | CUDA 线程局部内存          |
| `mem.const`  | CUDA / Wasm 常量内存       |

```rust
r2 = mem.shared.i32[r1]
r3 = mem.const.f32[r1 + 16]
```

在编译目标为 CPU（x86 / ARM / RISC-V）时，除了 `global`，其他地址空间会被拒绝。

## 对齐

Aperio **不**对内存访问的对齐做运行期强制。未对齐访问的行为是目标相关的——x86 允许（可能慢），ARM / RISC-V 有些指令会触发陷阱。详见 [38. 未定义行为](./38_undefined_behavior.md)。

想显式做可能未对齐的访问，用 **`mem.<T>.unaligned[...]`**：

```rust
r2 = mem.i32.unaligned[r1]               // 保证在任何平台上都不会因为未对齐而陷阱
```

编译器会在需要时拆成多次窄访问。

## `volatile` 内存

禁止编译器重排/合并/消除的访问用 **`mem.<T>.volatile[...]`**，详见 [10. volatile 内存](./10_volatile.md)。

## 与 `std/atomic` 的区别

`mem.*` 是**非原子**访问——多线程场景下有数据竞争就是 UB。想要原子性走 [`std/atomic`](./29_std_atomic.md)。
