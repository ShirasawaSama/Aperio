# 12. 内联函数

`inline` 修饰符告诉编译器：**在调用处直接展开这个函数的函数体**，而不是生成一条真正的 `call` 指令。

## 语法

在 `fn` 前面加上 `inline` 关键字：

```text
inline fn add(r1, r2) -> r0 {
    r0 = r1 + r2
}
```

可以与 `pub` / `export` 组合：

```text
inline pub fn fast_mul(r1, r2) -> r0 {
    r0 = r1 * r2
}
```

## 行为

`inline` 函数**没有独立的机器码实体**——编译器不会为它生成入口地址，调用点也不会出现 `call` 指令。函数体被直接嵌入到每一个调用处，类似于 C 的 `static inline` 或者宏展开（但保留了函数的类型语义）。

对下面这段代码：

```text
inline fn double(r1) -> r0 {
    r0 = r1 * 2
}

pub fn main() -> r0 {
    double(r1 = 10)
}
```

经过内联展开，相当于：

```text
pub fn main() -> r0 {
    r1 = 10
    r0 = r1 * 2       // double 的函数体
}
```

## ABI 约束

即使是 `inline` 函数，签名中的参数、返回值、`uses` 仍然要遵守——因为调用方写的还是 ABI 风格的代码（用命名参数调用）。内联只是省掉了跳转，**不改变寄存器使用规则**。

## 限制

`inline` 函数有以下限制：

### 不能取地址

```text
inline fn foo() -> r0 { r0 = 0 }

r1 = &foo        // 编译错误：inline 函数没有实体地址
```

没有实体就没有地址，这和 C 里对 `inline` 函数取地址的尴尬处境一样。

### 不能递归

```text
inline fn bad(r1) -> r0 {
    bad(r1 = r1)        // 编译错误：inline 函数不能直接或间接递归
}
```

展开递归函数会导致无限展开。

### 不能跨模块

`inline` 函数的定义必须在调用点可见。如果一个 `inline pub` 函数被另一个模块 `import`，它的函数体必须对导入方可见（具体机制详见 [18. 模块](./18_modules.md)）。

## 什么时候用 inline

- 极小的函数：`r0 = r1 + r2` 这种，函数调用开销比函数本身还大
- 性能热点中的辅助函数
- 编译期常量参数参与运算的场景（内联后可能被进一步常量折叠）

什么时候**不**用：

- 函数体很大，内联会显著增大代码体积
- 函数会被频繁递归调用
- 需要函数指针的场景
